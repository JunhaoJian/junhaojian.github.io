    • 2026.1 - NOW Smart Contract Vulnerability Detection for HK Web3 Market
      Python, LLM Multi-Agent Systems, Smart Contract Analysis, Solidity, n8n, Slither, Mythril, Research Writing
      Problem

      Hong Kong's Web3 ecosystem faces severe financial and systemic risks from smart contract vulnerabilities. Traditional automated tools suffer from high false positives, poor logic flaw coverage, and weak scalability; single-LLM audits lack evidence traceability and inconsistent results, failing to reliably secure DeFi protocols and cross-contract interactions.

      Data

      Used public vulnerability benchmarks: Web3Bugs, DeFiHackLabs, and GPTScan-Top200. The dataset includes labeled Ethereum smart contracts with typical flaws (reentrancy, access control, integer overflow, logical bugs) for standardized testing and false-positive validation.

      Approach

      Built an n8n-orchestrated multi-agent detection framework that fuses “hard” evidence from Slither/Mythril with “soft” semantic reasoning from GPT-4o. The workflow includes vulnerability triage, code-linked explanation, adversarial verification, evidence aggregation, and risk scoring to improve accuracy and reduce false alerts. Developed Solidity test contracts with intentional vulnerabilities for framework validation.

      Outcome & Impact

      Established a reproducible auditing benchmark with pinned environments to ensure stable testing. The multi-agent system outperforms tool-only and single-LLM baselines by lowering false positives and better identifying business-logic vulnerabilities. The solution provides explainable, traceable results to enhance security and stability for Hong Kong’s Web3 market. The project is ongoing, with plans for further optimization and real-world deployment.

      • Led project planning, task allocation & progress management
      • Led system design & architecture of multi-agent framework
      • Directed team execution & ensured delivery quality
      • Conducted static & dynamic smart contract detection and led research writing, analysis & reporting
    • 2024.10 - 2025.4 Social Media Sentiment Analysis System GitHub
      Python, Flask, Web Crawling, MySQL, Sentiment Analysis, LSTM, Data Visualization
      Problem

      Social media platforms generate massive unstructured text data, making it difficult to efficiently identify public sentiment. Traditional analysis methods suffer from low accuracy, poor scalability, and lack of intuitive visualization, which limits enterprises' precision marketing and governments' public opinion monitoring.

      Data

      Collected 89k+ Weibo hot-search data points (posts, comments, likes, reposts, user info) via web crawling. Used positive/negative sentiment corpus for model training, covering social hot topics and user-generated content.

      Approach

      Built a full-stack system with Flask as the web framework, Hadoop for distributed storage, and MySQL for data management. Performed data crawling, cleaning and preprocessing; developed a sentiment analysis model by fusing TextBlob and LSTM; realized interactive data visualization dashboards for result presentation.

      Outcome & Impact

      The system completed automated sentiment classification with a test accuracy of 74.58%. It supports real-time public opinion analysis and multi-dimensional data visualization, providing effective tools for marketing strategy and social opinion supervision.
